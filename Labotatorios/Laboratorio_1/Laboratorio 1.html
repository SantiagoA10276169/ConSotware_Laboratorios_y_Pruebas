  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

  • ¿Cuáles son las partes de un URL?

    • esta dividido en 4 partes

    • esquema: identifica protocolos
    • Host: es el indentificador de recursos
    • Via de acceso: identifica el recurso especifico al que el host quiere acceder
    • Serie de consulta: Proporciona una serie de informacion que el recuso utilizara para un propocito

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • GET: Solicitar datos de un servidor
    • HEAD: Solicitar las cabeceras de la respuesta sien el cuerpo
    • POST: Envia datos al servidor
    • PUT: Sirve para actualizar recusos o crear recursos
    • PATCH: Sire para aplicar modificaciones de recuros, pero solo envia los cambios
    • DELETE: Elimina recusos del servidor

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    • EL metodo de enviar formulario es via POST pues este permite enviar datos al cuerpo d ela solicitud , manteniendolo más seguro y el envio de muchos datos

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    • Utiliza GET, pues solicita al servidor los datos para mostrarlo en la pagina solicitada

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    • significa que la solicitud se completo con exito, por lo que no es un error

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    • El error 404 significa "No enonctrado" por lo que el servidor no pudo ecnontrar la solicitud, por lo que es responsabilidad del desarrollador determinar el porque del error

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    • El error 500 significa "error interno del servidor" significando que algo en el servidor, por lo que el desarrolador es el responsable corregir los fallos del servidor
